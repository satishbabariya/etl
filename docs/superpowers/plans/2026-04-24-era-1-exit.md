# Era I Exit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close out Era I by making the platform self-debuggable (Prometheus + Grafana), operable (terminate + status CLI), and extensible (hello-world connector + SDK docs), then prove it against a real developer-chosen Postgres via a dogfood script.

**Architecture:** A minimal RFC-15 *operational plane* only — the worker exposes `/metrics` on a side HTTP endpoint via `metrics-exporter-prometheus`, activities emit counters and histograms through the `metrics` facade. Prometheus scrapes the worker; Grafana renders a bundled dashboard JSON. The CLI gains two new verbs (`workflow terminate`, `pipeline status`) that call Temporal + the catalog directly — no new service, no customer-facing plane. A `hello-world-source` WASM example plus a focused `crates/connector-sdk/README.md` walk a new developer through authoring and building a connector. A `scripts/dogfood-real-db.sh` points at an arbitrary `DATABASE_URL` + table and runs a cursor-incremental replication end-to-end.

**Tech Stack:** Unchanged core (Rust 1.88, Temporal, Arrow 53, wasmtime 26, sqlx 0.8). Additions: `metrics = "0.23"` + `metrics-exporter-prometheus = "0.15"` for metrics, `axum = "0.7"` (thin HTTP wrapper) for the `/metrics` endpoint, `prom/prometheus` + `grafana/grafana` docker images in `docker-compose.yml`.

```rust
use axum::{routing::get, Router};
use metrics_exporter_prometheus::PrometheusHandle;
use std::net::SocketAddr;

/// Serves `/metrics` on `bind`. Spawns a tokio task; errors are logged
/// but do not bring down the worker.
pub fn spawn_metrics_endpoint(handle: PrometheusHandle, bind: SocketAddr) {
    let app = Router::new().route(
        "/metrics",
        get(move || {
            let h = handle.clone();
            async move { h.render() }
        }),
    );
    tokio::spawn(async move {
        match tokio::net::TcpListener::bind(bind).await {
            Ok(listener) => {
                tracing::info!(%bind, "metrics endpoint listening");
                if let Err(e) = axum::serve(listener, app).await {
                    tracing::error!(error = %e, "metrics endpoint exited");
                }
            }
            Err(e) => {
                tracing::error!(error = %e, %bind, "metrics endpoint bind failed");
            }
        }
    });
}
```

```rust
//! Periodic poll of Postgres `pg_replication_slots.confirmed_flush_lsn`
//! lag, emitted as a gauge per slot. Lives as a long-running tokio task
//! on the worker; not invoked per-activity.

use catalog::Catalog;
use std::sync::Arc;
use std::time::Duration;

pub fn spawn_slot_lag_poller(
    catalog: Arc<Catalog>,
    source_url_resolver: impl Fn(uuid::Uuid) -> Option<String> + Send + Sync + 'static,
    interval: Duration,
) {
    tokio::spawn(async move {
        loop {
            tokio::time::sleep(interval).await;
            // Fetch all active slots; ignore errors but keep the loop alive.
            let rows = match sqlx::query_as::<_, (uuid::Uuid, String)>(
                "SELECT pipeline_id, slot_name FROM cdc_slots WHERE state = 'active'",
            )
            .fetch_all(catalog.pool())
            .await
            {
                Ok(r) => r,
                Err(e) => {
                    tracing::warn!(error = %e, "slot-lag poller: catalog query failed");
                    continue;
                }
            };
            for (pipeline_id, slot_name) in rows {
                let Some(url) = source_url_resolver(pipeline_id) else { continue };
                match crate::connectors::postgres::cdc::slot::slot_lag_bytes(&url, &slot_name).await {
                    Ok(lag) => {
                        metrics::gauge!(
                            crate::metrics::CDC_SLOT_LAG_BYTES,
                            "pipeline_id" => pipeline_id.to_string(),
                        )
                        .set(lag as f64);
                    }
                    Err(e) => {
                        tracing::debug!(error = %e, slot = %slot_name, "slot-lag poll failed");
                    }
                }
            }
        }
    });
}
```

```rust
    let catalog_for_resolver = catalog.clone();
    let source_url_resolver = move |pid: uuid::Uuid| -> Option<String> {
        let cat = catalog_for_resolver.clone();
        // Sync lookup via block_in_place — the poller only calls us from
        // its own background tokio task.
        tokio::task::block_in_place(|| {
            tokio::runtime::Handle::current().block_on(async move {
                let row: Option<(serde_json::Value,)> = sqlx::query_as(
                    "SELECT c.config FROM pipelines p \
                     JOIN connections c ON c.connection_id = p.source_conn_id \
                     WHERE p.pipeline_id = $1",
                )
                .bind(pid)
                .fetch_optional(cat.pool())
                .await
                .ok()
                .flatten();
                row.and_then(|(cfg,)| {
                    cfg.get("url").and_then(|v| v.as_str()).map(|s| s.to_string())
                })
            })
        })
    };
    worker::cdc_monitor::spawn_slot_lag_poller(
        catalog.clone(),
        source_url_resolver,
        std::time::Duration::from_secs(15),
    );
```

`crates/cli/src/terminate.rs`:

```rust
use anyhow::Context;
use catalog::Catalog;
use temporalio_client::WorkflowTerminateOptions;
use worker::temporal::{make_client, TemporalConfig};

pub async fn terminate(workflow_id: String, reason: Option<String>) -> anyhow::Result<()> {
    let cfg = TemporalConfig::from_env()?;
    let client = make_client(&cfg).await?;
    let handle: temporalio_client::UntypedWorkflowHandle<_> =
        client.get_workflow_handle::<temporalio_client::UntypedWorkflow>(workflow_id.clone());
    handle
        .terminate(WorkflowTerminateOptions {
            reason: reason.clone().unwrap_or_else(|| "platform-cli-terminate".into()),
            ..Default::default()
        })
        .await
        .context("terminate_workflow_execution")?;
    println!("terminated workflow {}", workflow_id);

    // Best-effort catalog update: mark the matching run Failed if we can
    // find it via temporal_workflow_id.
    let db_url = std::env::var("DATABASE_URL").context("DATABASE_URL must be set")?;
    let catalog = Catalog::connect(&db_url).await?;
    let row: Option<(uuid::Uuid,)> = sqlx::query_as(
        "SELECT run_id FROM runs WHERE temporal_workflow_id = $1 ORDER BY started_at DESC LIMIT 1",
    )
    .bind(&workflow_id)
    .fetch_optional(catalog.pool())
    .await?;
    if let Some((rid,)) = row {
        let rid = common_types::ids::RunId::from_uuid_unchecked(rid);
        catalog
            .mark_run_failed(rid, &reason.unwrap_or_else(|| "terminated".into()))
            .await?;
        println!("marked run {} failed", rid);
    } else {
        println!("no matching runs row (workflow may be external)");
    }
    Ok(())
```

```rust
// Emits a fixed 3-row batch on the first call, then EOF.
//
// The Arrow IPC bytes in this file were generated by the Rust host — we
// encode them at runtime instead to keep the example self-contained.

wit_bindgen::generate!({
    path: "../../crates/connector-sdk/wit",
    world: "source-connector",
});

use exports::platform::connector::connector as guest;

struct HelloWorld;

impl guest::Guest for HelloWorld {
    fn discover(
        _conn: guest::ConnectionConfig,
        _source: guest::SourceConfig,
    ) -> Result<Vec<u8>, String> {
        // Arrow schema: id: Int64, greeting: Utf8 — hand-rolled IPC bytes.
        // Simpler: emit the same small IPC bytes for schema as for data,
        // since the host will re-derive schema from the first batch anyway.
        Ok(EMPTY_SCHEMA_IPC.to_vec())
    }

    fn read_batch(
        _conn: guest::ConnectionConfig,
        _source: guest::SourceConfig,
        cursor: Option<guest::CursorValue>,
        _batch_size: u32,
    ) -> Result<guest::ReadOutcome, String> {
        // Cursor is a simple "page number". Page 0 → three rows; page 1 → empty.
        let page: i64 = cursor
            .as_ref()
            .map(|c| c.value.parse().unwrap_or(0))
            .unwrap_or(0);
        if page >= 1 {
            return Ok(guest::ReadOutcome {
                batch_ipc: EMPTY_SCHEMA_IPC.to_vec(),
                new_cursor: cursor,
                is_final: true,
            });
        }
        Ok(guest::ReadOutcome {
            batch_ipc: HELLO_BATCH_IPC.to_vec(),
            new_cursor: Some(guest::CursorValue {
                kind: guest::CursorKind::Int64,
                value: "1".into(),
            }),
            is_final: true,
        })
    }
}

export!(HelloWorld);

// Pre-rendered Arrow IPC bytes. See README for how these were generated.
const EMPTY_SCHEMA_IPC: &[u8] = include_bytes!("../assets/empty-schema.arrows");
const HELLO_BATCH_IPC: &[u8] = include_bytes!("../assets/hello-batch.arrows");
```

```rust
// Not part of the guest build; invoked from inside the workspace via a throwaway bin.
fn main() {
    use arrow::array::{Int64Array, StringArray};
    use arrow::datatypes::{DataType, Field, Schema};
    use arrow::ipc::writer::StreamWriter;
    use arrow::record_batch::RecordBatch;
    use std::sync::Arc;

    let schema = Arc::new(Schema::new(vec![
        Field::new("id", DataType::Int64, false),
        Field::new("greeting", DataType::Utf8, false),
    ]));
    // Empty schema-only IPC
    {
        let mut buf = Vec::new();
        let mut w = StreamWriter::try_new(&mut buf, &schema).unwrap();
        w.finish().unwrap();
        std::fs::write("examples/hello-world-source/assets/empty-schema.arrows", buf).unwrap();
    }
    // Three-row batch
    {
        let ids = Int64Array::from(vec![1, 2, 3]);
        let greetings = StringArray::from(vec!["hello", "world", "platform"]);
        let batch = RecordBatch::try_new(
            schema.clone(),
            vec![Arc::new(ids), Arc::new(greetings)],
        )
        .unwrap();
        let mut buf = Vec::new();
        let mut w = StreamWriter::try_new(&mut buf, &schema).unwrap();
        w.write(&batch).unwrap();
        w.finish().unwrap();
        std::fs::write("examples/hello-world-source/assets/hello-batch.arrows", buf).unwrap();
    }
    println!("generated hello-world-source assets");
}
```

# connector-sdk — authoring source connectors

This crate defines the Rust-side `SourceConnector` trait and the WIT
interface for the Component Model. Phase I.3 established the
surface; Phase I.6 added an LSN cursor kind for CDC. Era I exit adds
this walkthrough.

Three ways to write a connector:

1. **Rust, in-process.** Implement the `SourceConnector` trait directly
   — fastest to iterate, lives inside the worker binary.
2. **Rust, WASM Component Model.** Implement the WIT interface, compile
   to `wasm32-wasip2`, precompile with `platform connector build`.
   Sandboxed, network-isolated, hot-swappable.
3. **Any language with wit-bindgen support.** Same WIT interface — use
   your language's wit-bindgen; the host doesn't care.

Era I exit ships only the Rust-WASM path as stable.

## The interface (`wit/source-connector.wit`)

Two functions, four records:

```wit
package platform:connector@0.1.0;

interface types {
  record connection-config { url: string }
  record source-config      { json: string }
  enum cursor-kind { int64, timestamp-tz, lsn }
  record cursor-value { kind: cursor-kind, value: string }
  record read-outcome { batch-ipc: list<u8>, new-cursor: option<cursor-value>, is-final: bool }
}

interface connector {
  use types.{...};
  discover: func(conn: connection-config, source: source-config) -> result<list<u8>, string>;
  read-batch: func(conn: connection-config, source: source-config, cursor: option<cursor-value>, batch-size: u32) -> result<read-outcome, string>;
}

world source-connector {
  import platform:connector-host/host;
  export platform:connector/connector;
}
```

- `read-batch` returns up to `batch-size` rows **strictly after**
  `cursor`, plus a new cursor of the *last* row and an `is-final`

  `log(level, message)` and `http-fetch` (sandboxed). See

```bash
#!/usr/bin/env bash
# Replicate a developer-chosen Postgres table to local Parquet using the
# cursor-incremental pipeline. Prints before/after row counts + size
# summary. Not a test — a demo.
#
# Usage:
#   scripts/dogfood-real-db.sh <source-pg-url> <schema.table> <cursor-column> [cursor-kind]
# Example:
#   scripts/dogfood-real-db.sh \
#     'postgres://me:pw@localhost:5432/mydb' public.events updated_at timestamp_tz

set -euo pipefail

if [[ $# -lt 3 ]]; then
  echo "usage: $0 <pg-url> <schema.table> <cursor-column> [cursor-kind=timestamp_tz]" >&2
  exit 1
fi

SOURCE_URL="$1"
QUALIFIED="$2"
CURSOR_COL="$3"
CURSOR_KIND="${4:-timestamp_tz}"
SCHEMA="${QUALIFIED%.*}"
TABLE="${QUALIFIED#*.}"

WORKDIR="./data/dogfood"
mkdir -p "$WORKDIR"

echo "→ source: $SOURCE_URL"
echo "→ target: $WORKDIR/<pipeline_id>/"

# 1. Probe the source — count rows so we know what "done" looks like.
SOURCE_COUNT=$(psql "$SOURCE_URL" -At -c "SELECT COUNT(*) FROM $QUALIFIED")
echo "→ source rows: $SOURCE_COUNT"

# 2. Write a one-shot YAML spec.
YAML_FILE="$(mktemp -t dogfood.XXXXX.yaml)"
cat > "$YAML_FILE" <<YAML
apiVersion: platform/v1
kind: Connection
metadata:
  name: dogfood-source
spec:
  connector_ref: postgres@0.1.0
  config:
    url: "$SOURCE_URL"
---
apiVersion: platform/v1
kind: Pipeline
metadata:
  name: dogfood-${TABLE}
spec:
  source:
    type: postgres
    schema: $SCHEMA
    table: $TABLE
    cursor_column: $CURSOR_COL
    cursor_kind: $CURSOR_KIND
    pk_columns: [id]
  destination:
    type: local_parquet
    base_path: "$WORKDIR"
  batch_size: 1000
  evolution_policy: propagate_additive
YAML

echo "→ applying pipeline spec"
cargo run --bin platform -- apply -f "$YAML_FILE" >/dev/null
PIPELINE_ID=$(cargo run --quiet --bin platform -- get pipeline dogfood-${TABLE} | grep -E '^name:|^pipeline_id:' | head -1)
echo "→ pipeline: $PIPELINE_ID"

# 3. Launch the worker if it isn't running.
if ! pgrep -f 'target/debug/worker' >/dev/null; then
  echo "→ starting worker"
  cargo build --bin worker >/dev/null
  ./target/debug/worker &
  WORKER_PID=$!
  trap "kill $WORKER_PID 2>/dev/null || true" EXIT
  sleep 3
fi

# 4. Run the pipeline.
cargo run --bin platform -- pipeline run dogfood-${TABLE}

# 5. Poll for completion via status CLI.
DEADLINE=$(( $(date +%s) + 120 ))
while true; do
  STATUS=$(cargo run --quiet --bin platform -- pipeline status dogfood-${TABLE} 2>/dev/null | grep latest_run_status || true)
  echo "   $STATUS"
  if echo "$STATUS" | grep -qE 'completed|failed'; then break; fi
  if [[ $(date +%s) -gt $DEADLINE ]]; then echo "timeout"; exit 2; fi
  sleep 3
done

# 6. Summarise the Parquet output.
OUT_DIR=$(find "$WORKDIR" -type d -name "$(echo "$STATUS" | head -c 0)" -print -quit || true)
PARQUET_COUNT=$(find "$WORKDIR" -name '*.parquet' | wc -l | tr -d ' ')
TOTAL_BYTES=$(du -sh "$WORKDIR" | awk '{print $1}')
echo "✔ landed $PARQUET_COUNT parquet files, $TOTAL_BYTES total, for $SOURCE_COUNT source rows"
```

## Appendix A — Operational notes

**Worker port collision.** `/metrics` binds to 9898 by default. Override
via `ETL_METRICS_BIND=0.0.0.0:<port>`. Tests don't exercise the endpoint.

**Grafana anonymous-admin.** The docker-compose config disables the
login form entirely — fine for localhost, not for anything reachable
from outside. Phase II.1 will lock this down via the multi-tenant auth
stack (RFC-19).

**Metric label cardinality.** `etl_cdc_slot_lag_bytes{pipeline_id=...}`
grows with the number of pipelines. For Era I scale this is fine; Phase
II should drop the label and use Prometheus relabeling if cardinality
becomes a problem.

**`metrics` crate renders via snapshot, not streaming.** Very large
counters/histograms can make `/metrics` render slow. Era I scale is
fine; we'll revisit if the dashboard ever times out.

**Dogfood script needs `psql` installed locally.** The script
`scripts/dogfood-real-db.sh` uses the host's `psql` for the pre-flight
row count. If unavailable, the script fails before running the pipeline
— by design. Install via `brew install postgresql` or similar.
